# Sova AI｜指标异动归因工作台

Sova AI 是一个面向中文业务分析 / 数据分析场景的 AI 指标异动归因工作台。

“Sova” 有猫头鹰与侦察的意象，代表从复杂业务数据中观察异常信号、追踪隐藏线索，并形成可验证的证据链。

它不是普通的 Chat with CSV 工具，也不是简单图表生成器。它的核心目标是：当用户遇到一个模糊的业务指标异动问题时，系统先通过多轮业务澄清，把问题转化为可执行的数据分析任务；随后引导用户上传数据，使用 Pandas + DuckDB 执行分析，并生成结果表、图表、证据链和报告草稿。

## 当前指标口径生成逻辑

- 前端已新增 rule-based / mock Metric Definition Generator。
- 系统会根据用户输入的业务问题识别指标类型，并动态生成前三张候选指标口径卡片。
- 第四张“自定义口径”卡片固定保留。
- 优惠券核销率只是一个内置识别模板。
- 当前支持的 mock 指标类型：
  - 优惠券 / 核销率
  - DAU / 活跃
  - 转化率
  - GMV / 销售额
  - 留存
  - 退款
  - 无法识别时使用通用业务指标口径
- 当前仍然不接真实 LLM，未来可以将该生成器替换或扩展为 OpenAI-compatible provider。

## Stage 2A 对比周期确认

- 用户确认指标口径后，主交互区会进入“对比周期确认”步骤。
- 对比周期选项使用交互式卡片，包含“本周 vs 上周”“本月 vs 上月”“活动期 vs 活动前”“自定义对比周期”。
- 选择“自定义对比周期”后，需要填写具体周期才算确认。
- 确认对比周期后，右侧“分析就绪面板”会推进到“分析维度”阶段，并将“对比周期”加入已确认信息。

## Stage 2B 分析维度多选

- 用户确认对比周期后，主交互区会进入“分析维度选择”步骤。
- 分析维度选项继续使用交互式卡片，支持多选、取消选择和自定义维度。
- 当前内置维度包括“用户类型”“地区 / 城市”“渠道来源”“商品 / 商家 / 内容”“时间粒度”“自定义维度”。
- 至少选择一个维度后，“确认分析维度”按钮才可用。
- 确认分析维度后，右侧“分析就绪面板”会保留在“分析维度”阶段，但状态会进入“近期变化因素确认”。

## Stage 2C 近期变化因素与数据准备

- 用户确认分析维度后，主交互区会进入“近期变化因素确认”步骤。
- 近期变化因素继续使用交互式卡片，支持多选、取消选择，并对“暂无明显变化”“不确定”做互斥处理。
- 确认变化因素后，右侧“分析就绪面板”会进入“数据准备”阶段。
- 数据准备阶段会展示“当前理解摘要”和“下一步需要的数据”提示。
- Stage 3A 已将原先的“下一阶段：上传数据”mock 按钮替换为真实上传入口。

## 前端运行方式

进入前端目录：

```bash
cd metricflow-ai/frontend
```

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

默认访问：

```text
http://localhost:3000
```

如需让前端调用其他后端地址，可以设置：

```bash
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```

## 后端运行方式

进入后端目录：

```bash
cd metricflow-ai/backend
```

创建并启用虚拟环境后安装依赖：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

启动 FastAPI：

```bash
uvicorn main:app --reload --port 8000
```

健康检查：

```text
http://localhost:8000/health
```

## Stage 3A 已完成内容

- 数据准备阶段已经提供真实“上传数据”入口，不再只是“下一阶段：上传数据”的 mock 按钮。
- 前端支持点击或拖拽上传 CSV、Excel（.xlsx / .xls）文件，当前支持单文件和多文件的表单结构。
- 后端新增 `POST /api/data/upload`，使用 Pandas 读取 CSV / Excel，并返回上传批次 `upload_id`。
- 后端会保存上传文件到 `backend/uploads/{upload_id}/`，该目录仅用于后续本地分析阶段衔接，不是数据库持久化。
- 后端会清洗字段名，保留原始字段名和清洗后字段名，并返回字段类型、缺失率、样例值和前 5 行样例数据。
- 前端会用卡片和表格展示“数据识别结果”，包括文件名、表名、行数、字段数量、字段列表、字段类型、缺失率和样例行。
- 系统会基于字段名规则判断“当前数据可支持的分析”，例如时间趋势分析、用户维度拆解、地区 / 城市拆解、渠道来源拆解、金额类指标分析、订单维度分析、优惠券领取 / 使用相关分析。
- 系统会结合用户已选择的分析维度提示“当前数据暂不支持的部分”，例如缺少城市、渠道、用户类型等字段。
- 上传成功后右侧“分析就绪面板”会补充“数据状态：已上传并完成字段识别”。
- “生成分析计划”按钮已经在 Stage 3B 升级为规则版真实功能，会基于业务澄清结果和上传 schema 生成结构化分析计划。

## Stage 3B 已完成内容

- 后端新增 `POST /api/analysis/plan`，基于业务问题、指标口径、对比周期、分析维度、近期变化因素和上传 schema 生成规则版分析计划。
- 分析计划会进行字段匹配，识别时间字段、指标相关字段、用户字段、订单字段、城市 / 地区字段、渠道字段、金额字段、优惠券字段、用户类型字段、商家 / 商品字段等。
- 分析计划会根据用户选择的分析维度生成分析步骤，例如按用户类型、地区 / 城市、渠道来源、商品 / 商家 / 内容或时间粒度拆解指标变化。
- 每个分析步骤会返回 `ready`、`partial` 或 `blocked` 状态，前端展示为“可执行”“部分支持”“暂不支持”。
- 前端“生成分析计划”按钮现在会调用后端接口，并在上传结果之后展示“分析计划”区域。
- “分析计划”区域包含分析目标、指标与周期、字段匹配情况、分析步骤、当前限制和下一步说明。
- 如果尚未上传数据，点击“生成分析计划”会提示“请先上传数据并完成字段识别。”
- Stage 3B 当时仍然不执行 DuckDB SQL；Stage 4A 已补充固定规则 DuckDB 基础执行。

## Stage 4A 已完成内容

- 后端新增 `POST /api/analysis/execute`，基于 `upload_id` 读取 Stage 3A 保存到 `backend/uploads/{upload_id}/` 的上传文件。
- 后端会重新用 Pandas 读取 CSV / Excel，清洗字段名，并将 DataFrame 注册到 DuckDB 内存连接。
- 当前执行层只使用固定规则 SQL，不做 AI 自动生成 SQL。
- 已支持第一批基础分析结果表：
  - 数据基础概览
  - 整体趋势分析
  - 用户维度分析
  - 地区 / 城市分析
  - 渠道分析
  - 金额分析
  - 优惠券相关分析
- 前端在“分析计划”下方新增“执行基础分析”按钮，点击后调用 `/api/analysis/execute`。
- 前端新增“分析执行结果”区域，会展示执行摘要、多个结果表、当前限制和注意事项。
- Stage 4B 已在结果表基础上补充基础可视化；Stage 5A 已补充规则版证据链；Stage 5B 已补充规则版报告草稿。

## Stage 4B 已完成内容

- 前端新增基于分析结果表的基础可视化，不新增后端接口。
- 新增 Recharts 作为轻量图表库。
- “分析执行结果”区域中新增“可视化分析”模块，用于展示基础趋势和维度差异。
- `overall_trend` 会生成“整体趋势变化”折线图。
- `user_breakdown`、`region_breakdown`、`channel_breakdown` 会生成基础柱状图。
- `amount_summary` 会展示金额指标卡片，包括总金额、平均金额、最大值、最小值。
- `coupon_summary` 会展示优惠券相关指标卡片，例如总记录数、有优惠券字段记录数、可能使用记录数、可能使用率。
- 如果当前结果表不适合生成图表，会显示“当前结果表暂不适合生成图表，可以继续查看表格结果。”
- 本阶段不做复杂 BI / Dashboard；Stage 5A 已补充规则版证据链；Stage 5B 已补充规则版报告草稿。

## Stage 5A 已完成内容

- 后端新增 `POST /api/analysis/evidence`，基于业务问题、指标口径、对比周期、分析维度、近期变化因素、分析计划和基础分析执行结果生成规则版证据链。
- 当前证据链仍然是 rule-based / mock generator，不接真实 LLM。
- 当前支持基于 `overall_trend`、`user_breakdown`、`region_breakdown`、`channel_breakdown`、`amount_summary`、`coupon_summary` 生成趋势、用户、地区、渠道、金额和优惠券相关证据。
- 前端在“可视化分析”之后新增“证据链”区域，点击“生成证据链”后展示证据链卡片。
- 每张证据链卡片包含标题、初步发现、数据证据、相关结果表 / 图表、可信程度和下一步验证建议。
- 如果尚未执行基础分析，点击“生成证据链”会提示“请先执行基础分析，再生成证据链。”。
- 证据链文案保持谨慎表达，只做当前数据支持的初步判断，不直接输出最终因果结论。

## Stage 5B 已完成内容

- 后端新增 `POST /api/analysis/report`，基于业务澄清结果、分析计划、基础分析执行结果和证据链生成规则版报告草稿。
- 报告草稿包含八个部分：分析背景、当前已确认信息、数据与字段情况、初步分析发现、证据链摘要、可能原因、当前限制、建议下一步验证。
- 前端在“证据链”之后新增“报告草稿”区域，点击“生成报告草稿”后展示文档预览。
- 如果尚未生成证据链，点击“生成报告草稿”会提示“请先生成证据链，再生成报告草稿。”。
- 报告草稿支持“复制报告内容”，复制后可以粘贴到文档中继续修改。
- 当前报告仍然是 rule-based / mock 草稿，不接真实 LLM，不导出 Word / PDF。
- 报告文案保持谨慎表达，避免直接输出最终因果结论。
